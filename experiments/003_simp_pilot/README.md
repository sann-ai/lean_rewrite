## Candidate 3 — `Fin.succAbove`

**File**: `Mathlib/Data/Fin/SuccPred.lean`
**Signals**: unfold_count=1, simp_lemma_count=23

### Definition

```lean
/-- `succAbove p i` embeds `Fin n` into `Fin (n + 1)` with a hole around `p`. -/
```

### `unfold` usage (downstream dependency)

```lean
-- Line 538

  rintro i j hij
  unfold succAbove at hij
```

### Why `@[simp]` might help

`succAbove` is defined as a conditional expression (`if ... then ... else ...`). Adding `@[simp]`
would let simp automatically unfold it, which could help proofs that currently do
`unfold succAbove at h; split_ifs`. The if-then-else structure also means simp's `split_ifs`
integration could further reduce proof complexity. This is a strong candidate because the
characterizing lemmas (`succAbove_of_castSucc_lt`, `succAbove_of_le_castSucc`) describe the
two branches exactly, and `@[simp]` would make them applicable without explicit `unfold`.
