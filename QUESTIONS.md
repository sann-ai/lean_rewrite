# QUESTIONS.md — エージェントから人間への問い合わせ

**Append-only**。エージェントが作業中に「短い回答で unblock できる判断」を人間に尋ねる場所。人間は各エントリの `**Answer:**` 欄に答えを追記する。回答後は次のエージェントが自動でタスクを再開する。

**この仕組みは `blocked: needs human` の代替ではなく補完**: 本当に行き詰まった(人間が長文で説明しないと解決できない)ものは従来通り `blocked: needs human`。QUESTIONS.md は「1〜2 文 or 選択肢で答えられる」範囲の問い向け。

---

## エージェント向けの使い方

- **本当に人間の判断が必要なときのみ** 使う。ルーティン的選択は自分で決めること
- 質問は **短く、選択肢を付けて** 書く(人間が短時間で答えられるよう)
- Q 番号は `Q001` から連番。ファイル末尾の既存番号を確認して次を付ける
- 質問を立てたら関連タスクの status を `blocked: awaiting Q0NN` に変更(planning 段階なら省略可)
- 1 セッションに投稿する質問は **最大 1 件**(複数ある場合は最もブロック度が高いものを選び、残りは次のエージェントへ)
- 質問を投稿したら、関連タスクの claim を解放(次のエージェントが別タスクに進めるように)

### フォーマット(必ずこの形式で append)

```
## Q0NN — <短いタイトル> — <agent-id> — <UTC ISO>

**Context**: 2〜3 文でやっていたこと
**Question**: 具体的に答えてほしいこと
**Options considered**:
- A: <案>
- B: <案>
- (任意) C: ...
**Blocker level**: low | medium | high
**Associated task**: T0NN または planning または (空欄)

**Answer**:

<!-- 人間が記入 -->
```

## 人間向けの使い方

- `**Answer:**` 行の直下に回答を書いて `git add QUESTIONS.md && git commit -m "answer: Q0NN" && git push`
- 回答は **可能な限り決定的に**(「状況による」だとエージェントが再質問しがち)。選択肢が提示されていれば `A` / `B` のように一語でも OK
- 質問をしたエージェントと別のエージェントが次 tick で拾うため、回答後に手動でタスクを触る必要はない。ただし特に急ぎなら、関連タスクの `status: blocked: awaiting Q0NN` を `open` に戻しておくとすぐ拾われる

## 未解決の質問を見つける方法

```bash
grep -B1 -A1 "^<!-- 人間が記入 -->" QUESTIONS.md  # 未回答の質問のみ抽出
```
